📨 聊天记录合并转发插件 for AstrBot
https://img.shields.io/badge/AstrBot-Plugin-blue
https://img.shields.io/badge/License-MIT-green

🚀 轻松实现跨群、跨好友的聊天记录合并转发，支持时间筛选与多目标转发的强大工具！

✨ 功能特点
✅ 将当前聊天的指定时间范围消息转发到 QQ 好友 / QQ 群

✅ 从指定 QQ 好友 / QQ 群拉取消息并转发到当前聊天

✅ 支持转发到多个目标（使用 | 分隔）

✅ 灵活的时间范围选择（相对时间 / 绝对时间）

✅ 完善的管理员权限系统（超级管理员可动态管理管理员列表）

✅ 原生支持合并转发样式（QQ 内置聊天记录样式）

📦 安装方法
1️⃣ 克隆仓库到 AstrBot 的插件目录：

bash
cd astrbot/plugins
git clone https://github.com/hanasaki/astrbot_plugin_chat_forward.git
2️⃣ 安装依赖：

bash
pip install -r requirements.txt
3️⃣ 重启 AstrBot 🎉

⚙️ 配置说明
配置文件位于 data/plugins/chat_forward/config.json，首次运行会自动生成。

json
{
  "admins": ["管理员QQ号"],
  "super_users": ["超级管理员QQ号"],
  "max_forward_count": 100,
  "enable_emoji": true,
  "forward_delay": 0.5
}
配置项	说明
admins	普通管理员列表（可使用转发命令）
super_users	超级管理员列表（可增删管理员）
max_forward_count	单次最大转发消息数
enable_emoji	是否启用表情包转发支持
forward_delay	转发时的延迟（秒），避免风控
🛠️ 使用命令
🔁 转发到目标（转去）
将当前聊天的历史消息转发给其他好友或群。

text
/转去 目标 时间范围
参数	说明
目标	QQ号（好友）或 g + 群号（群聊），多个目标用 | 分隔
时间范围	见下文【时间格式支持】
📌 示例：

命令	效果
/转去 123456 30分钟前	转发最近30分钟的消息给好友 123456
/转去 g789456 2小时前	转发最近2小时的消息给群 789456
/转去 123456|g789456 2024-01-01 到 2024-01-31	转发指定时间范围的消息给多个目标
📥 从源转发（转来）
从其他好友或群获取消息并转发到当前聊天。

text
/转来 源 时间范围
参数	说明
源	QQ号（好友）或 g + 群号（群聊）
时间范围	见下文【时间格式支持】
📌 示例：

命令	效果
/转来 123456 30分钟前	从好友 123456 获取最近30分钟的消息
/转来 g789456 2小时前	从群 789456 获取最近2小时的消息
👥 管理员管理
命令	说明	权限要求
/添加管理员 QQ号	添加普通管理员	仅超级管理员
/移除管理员 QQ号	移除普通管理员	仅超级管理员
/管理员列表	查看当前管理员列表	超级管理员 + 普通管理员
🕒 时间格式支持
相对时间（“X时间前”）
示例	含义
30分钟前	最近30分钟
2小时前	最近2小时
3天前	最近3天
1周前	最近1周
1月前	最近1个月（按30天计算）
绝对时间（“开始 到 结束”）
示例	含义
2024-01-01 到 2024-01-31	该日期范围内的全天消息
2024-01-01 10:00 到 2024-01-01 18:00	精确到小时的时段
💡 时间解析非常灵活，支持 年月日 时分秒 的多种组合。

⚠️ 注意事项
消息历史记录：需要 AstrBot 开启消息历史记录功能（或由适配器提供）。

QQ 风控：大量快速转发可能触发风控，建议保持合理 forward_delay。

合并转发兼容性：不同的 QQ 协议端（OneBot、GOCQ、QQBot 等）对合并转发消息的支持程度不同，请测试。

超级管理员：配置文件中 super_users 内的 QQ 号默认为超级管理员，无法被删除。
